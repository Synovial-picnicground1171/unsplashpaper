Features

Everything you need, nothing you don't

01

Automatic refresh

Choose your interval: every hour, every day, or anything in between. Your wallpaper updates itself in the background.

02

Skip or like

Not feeling the current photo? Skip to the next one. Love it? Like it to save the photographer and Unsplash link to your favorites.

03

Curated categories

Nature, architecture, space, minimalism, ocean, or any keyword you can think of. Change your category anytime from the settings.

04

Photographer credit

Every wallpaper notification shows the photographer's name. Open their Unsplash profile directly from the tray menu.

05

Start with your system

One toggle in the menu to launch UnsplashPaper when your computer starts. Works on Windows, macOS, and Linux. No admin rights required.

06

Minimal footprint

Only one image file on disk at any time. No background collection, no storage bloat. Just the current wallpaper.

Getting started

Up and running in five minutes

1

Get a free API key

You need to create a free Unsplash developer account to get an API key. This is a one-time step that takes a few minutes. See the detailed guide below.

2

Download and install

Grab the installer from the latest release page. Run it and click through the setup like any other program.

3

Configure on first launch

UnsplashPaper opens a settings window on first run. Paste your API key, choose a photo category and resolution, then hit Save.

4

That's it

The app lives in your system tray. Right-click the icon for all options: next wallpaper, like, settings, auto-start, and more.

The API key lets UnsplashPaper access photos from Unsplash on your behalf.
It is completely free and only needs to be done once. Follow these steps carefully.

  1. Create an Unsplash account

    Go to unsplash.com/join and sign up with your email address. If you already have an Unsplash account, skip to the next step.

  2. Go to the developer page

    Once logged in, go to unsplash.com/oauth/applications. This is where you manage your developer applications. Click the New Application button.

  3. Accept the terms

    Unsplash will show you their API usage guidelines. Check all the boxes to accept the terms, then click Accept terms.

  4. Fill in the application details

    You need to give your application a name and a short description. You can use anything you like, for example:

    Name: My Photo App

    Description: A personal app to browse photos

    Important: Do not use the words "wallpaper", "fond d'ecran", "background", or "screensaver" in the name or description. Unsplash automatically rejects applications containing these terms. Use a neutral name instead.

  5. Copy your Access Key

    After creating the application, you will see a page with your app details. Look for the field labeled Access Key (not "Secret Key"). It is a long string of letters and numbers. Select it and copy it.

    This is the key you will paste into UnsplashPaper on first launch.

FAQ

Common questions

Why do I need an API key?

Unsplash requires every application to authenticate with an API key to serve photos. Creating one is free and takes under a minute. The free tier allows 50 requests per hour, which is far more than you will ever need since UnsplashPaper only makes one request per wallpaper change.

Is my API key safe?

Your key is stored locally in a configuration file on your computer. It is never transmitted anywhere other than directly to the Unsplash API. The application is fully open source, so you can verify this yourself.

Does it use a lot of storage?

No. UnsplashPaper keeps exactly one image on disk: the current wallpaper. Each time a new photo is fetched, the previous file is replaced. Liked wallpapers are saved as metadata only, not as image files.

Can I change the category later?

Yes. Right-click the tray icon, open Settings, and change the category, resolution, or refresh interval at any time. The next wallpaper will use your new preferences.

Does it work offline?

UnsplashPaper needs an internet connection to fetch new photos. If you are offline, it simply keeps the current wallpaper until the next successful refresh.
